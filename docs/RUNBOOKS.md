# Runbooks

Runbooks should describe safe operational procedures at a high level. They should avoid sensitive implementation details in committed files.

## v0.1 Runbook Categories

- Device maintenance.
- Service restart planning.
- Backup verification planning.
- Access review planning.
- Incident notes template.

## Public Template Rules

- Use fake identifiers.
- Avoid real addresses and hostnames.
- Avoid private paths.
- Avoid secrets and credentials.
- Keep detailed local procedures in ignored files when they expose sensitive environment details.
